¿Para qué sirve esta página?

La página de Gestionar Formularios te permite crear, editar y administrar los formularios que se utilizan dentro del módulo de Procedimientos. Un formulario define los datos que se recopilan al iniciar un procedimiento o al ejecutarlo de forma independiente.

Cada formulario está compuesto por campos (preguntas) que pueden ser de distintos tipos: texto, números, fechas, selección de opciones, archivos, entre otros.

Requisito: Para acceder a esta página necesitas el permiso "Gestionar formularios" asignado a tu rol, o ser administrador del sistema.

Listado de Formularios

Al ingresar verás el listado de todos los formularios disponibles. Cada formulario muestra su nombre, descripción, cantidad de campos, modo de uso, fecha de creación y su estado (activo o inactivo).

Filtros
Búsqueda por texto

Escribe en el campo de búsqueda para encontrar formularios por nombre o descripción. Los resultados se filtran en tiempo real mientras escribes.

Filtro por estado

Usa el selector de estado para ver solo los formularios activos, solo los inactivos, o todos.

Acciones en cada formulario

Cada formulario tiene una fila de botones de acción:

  • Ver campos (ícono de ojo): Abre una ventana con la lista de campos del formulario y sus opciones, sin necesidad de entrar al modo de edición.
  • Editar (ícono de lápiz): Abre el formulario en modo de edición para modificar su nombre, descripción, campos o configuración.
  • Compartir (ícono de compartir): Solo aparece en formularios con modo "Independiente" o "Ambos". Genera enlaces y un código QR para distribuir el formulario.
  • Eliminar (ícono de papelera): Elimina el formulario. Si el formulario está asociado a un procedimiento, no se puede eliminar.
Importante: Eliminar un formulario es una acción irreversible. Si el formulario ya tiene datos asociados a procedimientos, el sistema no permitirá su eliminación.

Crear o Editar un Formulario

Al presionar el botón "Nuevo Formulario" (o al editar uno existente), se abre una ventana con tres secciones principales: información general, modo de uso y campos del formulario.

Información General
Nombre del formulario

Es obligatorio. Debe ser descriptivo para que se identifique fácilmente. Por ejemplo: "Solicitud de Permiso", "Reporte de Novedad", "Inscripción a Evento".

Descripción

Opcional. Puedes agregar un texto explicativo sobre el propósito del formulario. Es útil cuando hay muchos formularios y necesitas diferenciarlos rápidamente.

Estado

Un formulario puede estar Activo (disponible para uso) o Inactivo (no se puede usar pero se conserva para referencia). Al crear un formulario nuevo, queda activo por defecto.

Modo de Uso

Esta configuración define cómo se utilizará el formulario. Hay tres opciones:

Solo para procedimientos

El formulario solo se usa como paso inicial de un procedimiento. No se puede ejecutar de forma independiente. Es la opción predeterminada.

Solo ejecución independiente

El formulario se ejecuta por sí solo, sin estar asociado a un procedimiento. Puede ser interno (requiere que el usuario inicie sesión) o público (cualquier persona puede llenarlo sin necesidad de cuenta).

Ambos

El formulario puede usarse tanto dentro de procedimientos como de forma independiente. Combina las dos opciones anteriores.

Nota: Si seleccionas "Solo ejecución independiente" o "Ambos", aparecerán opciones adicionales para configurar el tipo de acceso (interno o público) y un email de notificación opcional que recibirá un aviso cada vez que alguien complete el formulario.
Opciones de ejecución independiente

Estas opciones solo aparecen cuando el modo de uso incluye ejecución independiente:

Acceso Interno

Solo pueden llenar el formulario las personas que tengan una cuenta en SchoolNet e inicien sesión. El sistema registra automáticamente quién lo completó.

Acceso Público

Cualquier persona puede llenar el formulario sin necesidad de cuenta ni inicio de sesión. Incluye un captcha de seguridad para evitar envíos automáticos. Es ideal para formularios dirigidos a familias, proveedores o personas externas al colegio.

Email de notificación

Opcionalmente puedes ingresar una dirección de correo electrónico que recibirá una notificación cada vez que alguien complete el formulario.

Compartir un formulario independiente

Para formularios con modo "Independiente" o "Ambos", el botón de compartir (ícono ) abre una ventana con:

  • Enlace interno: URL para usuarios que tienen cuenta en SchoolNet. Requiere iniciar sesión.
  • Enlace público: URL que cualquier persona puede abrir sin cuenta (solo si el formulario está configurado como público).
  • Código QR: Se genera automáticamente a partir del enlace principal. Puedes imprimirlo o compartirlo para que las personas escaneen y accedan directamente al formulario.
Consejo: Para copiar un enlace, presiona el botón de copiar (ícono de portapapeles) al lado del campo. El botón cambiará brevemente a un check verde confirmando que se copió.

Campos del Formulario

Los campos son las preguntas o datos que se recopilan. Un formulario debe tener al menos un campo para poder guardarse. Para agregar un campo, presiona el botón "Agregar Campo" dentro de la ventana de edición del formulario.

Tipos de campo disponibles
Texto corto

Para respuestas breves como nombres, códigos o datos simples. Permite agregar validaciones opcionales de formato: email, teléfono o URL.

Texto largo (Área de texto)

Para respuestas extensas como descripciones, observaciones o justificaciones. También permite validación de formato.

Número

Para valores numéricos. Permite definir un valor mínimo y un valor máximo como validación.

Fecha

Muestra un selector de fecha con calendario.

Selección (Dropdown)

Muestra una lista desplegable con opciones predefinidas. El usuario selecciona una. Debes agregar al menos una opción al crear el campo.

Selección única (Radio buttons)

Similar a Selección, pero las opciones se muestran todas visibles al mismo tiempo como botones de radio. El usuario elige una.

Casilla de verificación (Checkbox)

Una casilla que el usuario marca o desmarca. Se usa para aceptaciones, conformidades o verificaciones. Debes definir el texto que aparece junto a la casilla (por ejemplo: "Acepto los términos y condiciones").

Archivo

Permite al usuario adjuntar un archivo. Puedes configurar el tamaño máximo (en MB) y los tipos de archivo permitidos (por ejemplo: pdf, jpg, png).

SI/NO

Un campo simple con dos opciones: Sí o No.

Selección desde tabla del sistema

Muestra como opciones los registros de una tabla interna del sistema. Las tablas disponibles incluyen: Áreas Académicas, Cursos, Familias, Géneros, Grados, Roles de Trabajo, Secciones, Estudiantes y Trabajadores. Las opciones se actualizan automáticamente según los datos del sistema.

Configuración de cada campo

Al agregar o editar un campo, además del nombre y tipo, puedes configurar las siguientes opciones:

Campo obligatorio

Si marcas esta casilla, el usuario no podrá enviar el formulario sin completar este campo. Los campos obligatorios se identifican con un asterisco rojo (*) en el formulario.

Pre-llenado automático

Permite que el campo se complete automáticamente con información del usuario que está llenando el formulario. Las opciones disponibles son: nombre del usuario, email del usuario o fecha actual. Es útil para campos como "Solicitante" o "Fecha de solicitud", ya que evita que el usuario tenga que escribirlos manualmente.

Texto de ayuda (placeholder)

Un texto gris que aparece dentro del campo cuando está vacío, como guía para el usuario. Por ejemplo: "Ingrese su nombre completo" o "Seleccione una fecha".

Texto de ayuda (descripción)

Un texto explicativo que aparece debajo del campo. Útil para dar instrucciones más detalladas sobre qué información se espera o cómo debe completarse.

Lógica Condicional

La lógica condicional permite que un campo solo se muestre cuando otro campo tiene un valor específico. Esto es útil para formularios donde ciertas preguntas solo aplican en determinados casos.

Ejemplo: Tienes un campo "¿Requiere transporte?" con opciones "Sí" y "No". Puedes crear un segundo campo "Dirección de recogida" que solo aparezca cuando el usuario seleccione "Sí" en el primer campo.

Para configurar una condición:

  1. Al crear o editar un campo, marca la casilla "Este campo depende de otro campo".
  2. Selecciona el campo del que depende (solo aparecen campos de tipo Selección o Radio buttons que estén antes en el orden).
  3. Selecciona el valor específico que debe tener ese campo para que el campo actual se muestre.
Restricción: Un campo condicional solo puede depender de campos que estén antes que él en el orden del formulario, y solo de campos tipo Selección o Radio buttons (porque tienen opciones definidas).
Gestionar los campos del formulario

Una vez agregados los campos, puedes organizarlos y modificarlos desde la lista que aparece en la ventana de edición del formulario:

Reordenar

Usa los botones de flecha arriba y abajo para cambiar la posición de un campo. El orden en que aparecen aquí es el mismo orden en que se mostrarán al usuario que llene el formulario.

Editar

Presiona el botón de lápiz para abrir el campo en modo de edición y modificar cualquiera de sus propiedades.

Eliminar

Presiona el botón de papelera para quitar un campo del formulario. Si el campo ya tiene respuestas registradas en el sistema, no se eliminará sino que se desactivará para conservar los datos históricos.

Nota: Los cambios en los campos no se guardan hasta que presiones el botón "Guardar Formulario" en la parte inferior de la ventana. Si cierras la ventana sin guardar, los cambios se perderán.
Opciones de selección

Para campos de tipo Selección (Dropdown) y Radio buttons, necesitas definir las opciones que el usuario podrá elegir. Al configurar el campo:

  1. Presiona "Agregar Opción".
  2. Escribe el nombre de la opción (lo que verá el usuario).
  3. El sistema genera automáticamente un valor interno a partir del nombre.
  4. Repite para cada opción que necesites.

Las opciones se pueden reordenar con las flechas o eliminar con el botón de papelera. Al igual que con los campos, si una opción ya tiene respuestas asociadas, se desactivará en lugar de eliminarse.

Preguntas Frecuentes

¿Puedo editar un formulario que ya está en uso por un procedimiento?

Sí, puedes editar el nombre, la descripción, el estado y agregar nuevos campos. Sin embargo, ten en cuenta que los campos que ya tienen respuestas registradas no se pueden eliminar (se desactivan). Los cambios aplicarán a las nuevas solicitudes, no a las que ya están en curso.

¿Puedo eliminar un formulario que tiene un procedimiento asociado?

No. El sistema verifica si el formulario está vinculado a algún procedimiento antes de permitir su eliminación. Si necesitas dejar de usarlo, cámbialo a estado "Inactivo" en lugar de eliminarlo.

¿Qué pasa si cambio el tipo de un campo después de crearlo?

Al editar un campo puedes cambiar su tipo, pero esto puede afectar las validaciones y opciones configuradas. Por ejemplo, si cambias de "Selección" a "Texto corto", se perderán las opciones definidas. Revisa la configuración completa después de cambiar el tipo.

¿Cuántos campos puede tener un formulario?

No hay un límite técnico, pero se recomienda mantener los formularios concisos y enfocados. Un formulario con demasiados campos puede resultar tedioso para el usuario y aumentar el riesgo de abandono.

¿Qué es el campo "Selección desde tabla del sistema"?

Es un tipo especial que muestra como opciones los registros existentes en el sistema. Por ejemplo, si seleccionas la tabla "Trabajadores", el campo mostrará la lista de todos los trabajadores registrados. Las opciones se actualizan automáticamente cuando se agregan o modifican registros en el sistema.

¿Cómo comparto un formulario público con personas externas?

Primero asegúrate de que el modo de uso sea "Solo ejecución independiente" o "Ambos", y que el tipo de acceso sea "Público". Luego presiona el botón de compartir en el listado. Copia el enlace público o descarga el código QR para distribuirlo por correo, impreso o en redes.
