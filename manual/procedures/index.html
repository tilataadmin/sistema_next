¿Qué es el módulo de Procedimientos?

El módulo de Procedimientos permite gestionar flujos de trabajo institucionales de principio a fin. Desde la creación de formularios y la definición de pasos con responsables, hasta la ejecución, seguimiento y análisis de cada solicitud.

Al ingresar al módulo verás una página con tarjetas que representan cada funcionalidad disponible. Solo se muestran las funcionalidades a las que tienes acceso según los permisos asignados a tu rol.

Permisos: Cada funcionalidad requiere un permiso específico. Si no ves alguna tarjeta, significa que tu rol no tiene asignado el permiso correspondiente. Contacta al administrador del sistema si necesitas acceso.

Funcionalidades disponibles

El módulo está organizado en 8 funcionalidades. A continuación se describe cada una con su propósito y el permiso que requiere.

Configuración

Funcionalidades para crear y administrar la estructura de formularios y procedimientos.

Gestionar Formularios

Crea y configura los formularios que se usan para recopilar información. Define campos de distintos tipos (texto, número, fecha, selección, archivos, etc.), configura lógica condicional, pre-llenado automático y modos de uso (procedimientos, independiente o ambos).

Permiso: "Gestionar formularios"
Gestionar Procedimientos

Diseña los flujos de trabajo: crea procedimientos, define los pasos en secuencia, asigna responsables, configura bifurcaciones (caminos alternativos), pasos paralelos (múltiples tareas simultáneas), envío de correos automáticos, plazos SLA y valida que el flujo esté completo antes de activarlo.

Permiso: "Gestionar procedimientos"
Ejecución

Funcionalidades para iniciar procedimientos y llenar formularios.

Ejecutar Procedimiento

Inicia un procedimiento llenando el formulario asociado. El sistema crea automáticamente la solicitud, genera la primera tarea para el responsable y envía notificaciones por correo. Solo muestra procedimientos internos y activos.

Permiso: "Ejecutar procedimientos"
Llenar Formulario

Completa formularios independientes que no están asociados a un flujo de trabajo con pasos. La respuesta queda registrada inmediatamente como completada. Útil para encuestas internas, registros puntuales o solicitudes simples.

Permiso: "Ejecutar formularios"
Seguimiento

Funcionalidades para consultar el estado y avance de solicitudes.

Mis Solicitudes

Consulta el estado de los procedimientos que iniciaste. Incluye un timeline visual con el avance paso a paso, los datos del formulario inicial y la opción de cancelar solicitudes en curso. Si eres creador de un procedimiento, también ves las solicitudes de otras personas.

Permiso: "Ver mis solicitudes"
Consultar Registros

Vista global de todas las solicitudes del sistema con filtros avanzados (procedimiento, estado, SLA, solicitante, fechas). Incluye indicadores de cumplimiento SLA con semáforo, historial de pasos, vista agregada con estadísticas por campo y exportación a CSV.

Permiso: "Consultar todos los registros"
Consultar Respuestas

Busca y filtra las respuestas de formularios por el contenido de cada campo. Genera filtros dinámicos según el tipo de campo (texto, número, fecha, selección, etc.). Muestra los datos en tabla con vista previa de 3 campos y permite exportar todos los campos a CSV.

Permiso: "Consultar respuestas de formularios"
Análisis

Funcionalidades para monitorear el rendimiento de los procedimientos.

Dashboard

Panel de control con indicadores clave: procesos activos, completados, tiempo promedio, cumplimiento SLA, tareas vencidas. Incluye gráficos de tendencia, distribución por estado, detección de cuellos de botella, análisis por paso, rendimiento por responsable, modo comparación y exportación a PDF y Excel.

Permiso: "Ver dashboard de procedimientos"

Flujo general de trabajo

Para entender cómo se relacionan las funcionalidades entre sí, este es el flujo típico de trabajo con el módulo:

  1. Crear formulario: En "Gestionar Formularios" defines los campos que necesitas recopilar.
  2. Crear procedimiento: En "Gestionar Procedimientos" creas el procedimiento, le asocias el formulario y configuras los pasos con sus responsables y plazos.
  3. Activar: Validas el flujo y activas el procedimiento para que esté disponible.
  4. Ejecutar: Desde "Ejecutar Procedimiento" (o mediante enlace público si es externo), alguien llena el formulario e inicia la solicitud.
  5. Seguir: El solicitante consulta el avance en "Mis Solicitudes". Los administradores revisan en "Consultar Registros".
  6. Analizar: El "Dashboard" muestra métricas de rendimiento, cumplimiento SLA y cuellos de botella.
Consejo: Si solo necesitas recopilar información sin un flujo de aprobación, puedes usar "Llenar Formulario" directamente. No necesitas crear un procedimiento con pasos para eso.

Resumen de Permisos

Cada funcionalidad está protegida por un permiso específico. Los administradores del sistema (Super Admin) ven todas las funcionalidades sin restricción. Para el resto de usuarios, la tabla resume qué permiso necesita cada función:

Permisos de configuración

"Gestionar formularios" y "Gestionar procedimientos" son permisos de administración. Normalmente se asignan solo a coordinadores o responsables de definir los procesos institucionales.

Permisos de ejecución

"Ejecutar procedimientos" y "Ejecutar formularios" son permisos operativos. Se asignan a las personas que necesitan iniciar solicitudes o llenar formularios en su trabajo diario.

Permisos de consulta

"Ver mis solicitudes" es un permiso básico para cualquier usuario que ejecute procedimientos. "Consultar todos los registros" y "Consultar respuestas de formularios" son permisos avanzados para supervisores o coordinadores. "Ver dashboard de procedimientos" es para análisis gerencial.

Importante: Si no ves una funcionalidad en la página del módulo, no significa que no exista. Simplemente tu rol no tiene el permiso necesario. Contacta al administrador del sistema para solicitar acceso.

Preguntas Frecuentes

¿Qué necesito para empezar a usar procedimientos?

Como mínimo necesitas: crear un formulario (Gestionar Formularios), crear un procedimiento asociado a ese formulario (Gestionar Procedimientos), configurar al menos un paso con un responsable, validar el flujo y activar el procedimiento. Después ya se puede ejecutar.

¿Cuál es la diferencia entre un formulario y un procedimiento?

Un formulario define qué datos se recopilan (campos, tipos, validaciones). Un procedimiento define qué se hace con esos datos (pasos, responsables, plazos, bifurcaciones). Un procedimiento siempre tiene un formulario asociado, pero un formulario puede usarse también de forma independiente sin procedimiento.

¿Puedo tener procedimientos internos y externos?

Sí. Los internos solo los pueden iniciar usuarios autenticados del sistema. Los externos generan un enlace público que cualquier persona puede usar sin cuenta (ideal para formularios dirigidos a familias o proveedores). Ambos tipos se gestionan y consultan de la misma forma.

¿Las tareas de los procedimientos aparecen en el panel de tareas general?

Sí. Cuando un procedimiento avanza a un paso con responsable, se crea automáticamente una tarea asignada a esa persona. La tarea aparece en su panel de tareas con el contexto del procedimiento y un botón para ver los detalles de la solicitud.

¿Qué pasa si un responsable no completa su tarea a tiempo?

Si el paso tiene un SLA definido, la tarea se marca como vencida. Esto se refleja en el Dashboard de Procedimientos como tarea vencida y afecta el indicador de cumplimiento SLA. El sistema no bloquea el procedimiento automáticamente, pero los administradores pueden detectar la demora.

¿Puedo usar el módulo solo para formularios, sin procedimientos?

Sí. Puedes crear formularios en modo "Solo ejecución independiente" y usarlos desde "Llenar Formulario". Las respuestas se consultan desde "Consultar Respuestas". No necesitas crear ningún procedimiento para esto.
